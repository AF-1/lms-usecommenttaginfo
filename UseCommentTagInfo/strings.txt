	EN	About

PLUGIN_USECOMMENTTAGINFO_SETTINGS_SONGDETAILS_ABOUT_TEXT
	EN	<b>Use (key)words in a track's comment tag to display the string you set here on the song details information page.</b><br><br>Examples:<br><br>#1<br>You would like to include the record label in the <i>song details information page</i>. Add words like DG, HM or Stax to the comment tag of your tracks. Then set the <b>Search String</b> to any of those words, set the <b>Menu Item Name</b> to "Record label" and the <b>Displayed Item String</b> to the full name of the record label.<br><br>#2<br>Not all <i>live</i> or <i>best of</i> albums include "live" or "best of" in their album title.<br>Just include the word "LIVE" or "BESTOF" in the comment tag of those tracks, set <b>Search String</b> to "LIVE" or "BESTOF", <b>Menu Item Name</b> to "Live" or "Best of", and <b>Displayed Item String</b> to "yes" or "✔︎".<br><br><br><b>Notes:</b><br><br>- To avoid problems <b>don't use special characters or spaces in your keywords</b> and keep them short.<br>- You can use unicode <b>symbols</b> in displayed strings. <b>Hex code</b> works. Example: the hex code of a <b>black heart</b> is <xmp style="display: inline-block;font-weight:bold;">&#x2665;</xmp> and would be displayed as &#x2665;. For more unicode characters look <a href="https://www.rapidtables.com/web/html/html-codes.html">here</a> or <a href="https://codepoints.net/U+2665">here</a>.<br>- About the <b>position</b> of track information: in the <b>LMS web UI</b> there's actually no <b>"More Info" menu</b>, rather its content is displayed as part of the song details page. On piCorePlayer, Squeezebox Touch or in the Material skin web UI the "More Info" menu is actually a menu you can enter.<br><br>- If any of your displayed strings includes a character that's not part of a device's default font it won't be displayed. Either change the character or change the device's font.
